2 Перспективные форматы

2.1 VVC/H.266 (Versatile Video Coding)

H.266/VVC представляет новейшее поколение стандартов видеокодирования, утвержденное ITU-T и ISO/IEC в 2020 году. Этот стандарт нацелен на эффективную работу с контентом 4K, 8K и даже более высоких разрешений, обеспечивая значительные улучшения в эффективности сжатия по сравнению с предыдущими поколениями.

2.1.1 Новые технологии

VVC включает множество инновационных технологий кодирования:

  1. Квадратичная древовидная структура с тройным разбиением: более гибкое разделение блоков
  2. Множественные типы трансформации: DCT-II, DST-VII, DCT-VIII для различных типов контента
  3. Улучшенное внутрикадровое предсказание: 67 угловых режимов и новые неугловые режимы
  4. Аффинная компенсация движения: учет вращения, масштабирования и сдвига объектов
  5. Bilateral matching: двустороннее сопоставление для улучшения предсказания
  6. Адаптивное петлевое фильтрование: ALF для минимизации артефактов сжатия

VVC революционизирует традиционный подход к видеокодированию через внедрение адаптивных алгоритмов, способных анализировать характеристики контента в реальном времени. Система множественных типов трансформации позволяет автоматически выбирать наиболее подходящий математический аппарат для каждого блока изображения, что особенно эффективно для сложных текстур и переходных областей.

Аффинная компенсация движения представляет качественный скачок в обработке динамических сцен, позволяя точно отслеживать вращение, масштабирование и деформацию объектов. Это критично для современного контента с активной камерной работой, спортивных трансляций и игрового видео.

2.1.2 Улучшения эффективности

VVC демонстрирует существенные улучшения в эффективности сжатия:

  1. Сжатие: до 50% лучше H.265/HEVC при том же качестве
  2. 4K и 8K контент: особенно эффективен для сверхвысоких разрешений
  3. HDR поддержка: улучшенная обработка расширенного динамического диапазона
  4. 360-градусное видео: специальные инструменты для VR контента
  5. Разноплановый контент: оптимизация для различных типов видео

Тестирование VVC в реальных условиях показывает, что наибольший выигрыш достигается именно на контенте сверхвысоких разрешений, где традиционные методы H.265 начинают показывать свои ограничения. Для 8K контента экономия битрейта может достигать 60%, что делает передачу такого видео по существующим сетевым каналам технически и экономически целесообразной.

Особенно впечатляющие результаты VVC демонстрирует при работе с HDR контентом и расширенными цветовыми пространствами, где сложность цветовых переходов традиционно требовала высоких битрейтов. Новые алгоритмы квантизации учитывают особенности человеческого восприятия в условиях высокого динамического диапазона.

2.1.3 Совместимость и внедрение

Текущее состояние внедрения VVC:

  1. Аппаратная поддержка: первые чипы появились в 2022-2023 годах
  2. Программное декодирование: высокие вычислительные требования
  3. Телевещание: планируется внедрение в 2025-2027 годах
  4. Потоковые сервисы: экспериментальное использование
  5. Патентные вопросы: сложная ситуация с лицензированием

Внедрение VVC сталкивается с классическими проблемами новых стандартов видеокодирования - высокими вычислительными требованиями и сложностью патентного ландшафта. В отличие от предыдущих поколений, где переход происходил относительно плавно, VVC требует значительных инвестиций в новое аппаратное обеспечение из-за кардинально усложнившихся алгоритмов.

Ключевым фактором успеха VVC станет скорость появления доступных аппаратных декодеров в массовых устройствах. Производители чипов уже анонсировали поддержку VVC в новых поколениях мобильных процессоров и телевизионных SoC, что должно ускорить внедрение стандарта в потребительской электронике к 2025-2026 годам.

2.2 EVC (Essential Video Coding)

EVC - стандарт MPEG, разработанный как альтернатива VVC с особым упором на снижение патентных рисков. Стандарт предлагает два профиля с разными уровнями патентной нагрузки, позволяя разработчикам выбирать подходящий компромисс между эффективностью и стоимостью лицензирования.

2.2.1 Стандарт MPEG

EVC утвержден как стандарт ISO/IEC 23094 в 2020 году:

  1. Разработка: ISO/IEC MPEG совместно с участниками индустрии
  2. Цель: создание стандарта с предсказуемыми патентными обязательствами
  3. Подход: разделение на базовый и расширенный профили
  4. Временные рамки: разработка началась в 2017 году

Разработка EVC была инициирована растущими опасениями индустрии относительно усложнения патентного ландшафта современных видеокодеков. В отличие от предыдущих стандартов MPEG, где патентные вопросы решались post-factum, EVC изначально проектировался с учетом интеллектуальной собственности и лицензионных рисков.

Уникальный подход EVC заключается в создании "патентной прозрачности" - каждая технология в стандарте имеет четкую классификацию по степени патентного риска, что позволяет разработчикам принимать обоснованные решения о компромиссах между функциональностью и стоимостью лицензирования.

2.2.2 Базовый и основной профили

EVC предлагает гибкий подход к лицензированию:

Baseline Profile

  1. Использует только технологии, доступные в H.264
  2. Отсутствие новых патентных ограничений
  3. Эффективность сравнима с H.264 High Profile
  4. Подходит для бюджетных проектов

Main Profile

  1. Включает новые технологии для повышения эффективности
  2. Умеренные патентные обязательства
  3. До 25% лучше H.264 по эффективности
  4. Конкурентоспособен с H.265 в базовых сценариях

Двухуровневая архитектура EVC отражает реалии современного видеорынка, где сосуществуют массовые приложения с жесткими ограничениями по стоимости и премиум-сегмент с требованиями к максимальной эффективности. Baseline Profile обеспечивает безопасный переход для разработчиков, опасающихся патентных рисков, в то время как Main Profile предлагает современные возможности с контролируемыми обязательствами.

Возможность динамического переключения между профилями в рамках одного потока открывает новые сценарии использования, где критичные сегменты видео могут кодироваться в Main Profile для максимального качества, а остальной контент - в Baseline для минимизации рисков.

2.2.3 Коммерческое применение

EVC привлекателен для коммерческих решений:

  1. Предсказуемые расходы: четкие лицензионные условия
  2. Гибкость выбора: возможность выбора уровня технологий
  3. Промышленная поддержка: Samsung, Huawei, MediaTek и другие
  4. Целевые рынки: мобильные устройства, видеонаблюдение, OTT
  5. Внедрение: первые продукты появились в 2021-2022 годах

Стратегия EVC заключается в предоставлении разработчикам четкого выбора между стоимостью лицензирования и техническими возможностями. Baseline Profile позволяет избежать новых патентных рисков, используя только проверенные временем технологии H.264, в то время как Main Profile добавляет современные инструменты сжатия с контролируемыми лицензионными обязательствами.

Особенно привлекательным EVC выглядит для развивающихся рынков и стартапов, где предсказуемость расходов на лицензирование критична для бизнес-планирования. Крупные производители электроники рассматривают EVC как страховку против потенциальных патентных споров и возможность диверсифицировать технологические риски.

2.3 Экспериментальные разработки

Помимо стандартизированных кодеков, активно ведутся исследования в области применения искусственного интеллекта и машинного обучения для видеокодирования. Эти технологии обещают революционные изменения в эффективности сжатия и качестве видео.

2.3.1 Машинное обучение в кодировании

Применение ML-технологий в видеокодировании открывает новые возможности:

  1. Предсказание движения: нейронные сети для улучшения компенсации движения
  2. Режимы кодирования: автоматический выбор оптимальных параметров
  3. Пост-обработка: улучшение качества после декодирования
  4. Адаптивная квантизация: учет особенностей восприятия человека
  5. Супер-разрешение: повышение разрешения при декодировании

Двухуровневая архитектура EVC отражает реалии современного видеорынка, где сосуществуют массовые приложения с жесткими ограничениями по стоимости и премиум-сегмент с требованиями к максимальной эффективности. Baseline Profile обеспечивает безопасный переход для разработчиков, опасающихся патентных рисков, в то время как Main Profile предлагает современные возможности с контролируемыми обязательствами.

Возможность динамического переключения между профилями в рамках одного потока открывает новые сценарии использования, где критичные сегменты видео могут кодироваться в Main Profile для максимального качества, а остальной контент - в Baseline для минимизации рисков.

2.3.2 Нейронные сети

Различные архитектуры нейронных сетей находят применение в видеокодировании:

  1. Сверточные сети (CNN): для пространственного предсказания и фильтрации
  2. Рекуррентные сети (RNN/LSTM): для временного предсказания
  3. Автоэнкодеры: для прямого сжатия изображений и видео
  4. Генеративно-состязательные сети (GAN): для улучшения качества
  5. Трансформеры: для глобального контекста в видео

Интеграция различных архитектур нейронных сетей в единую систему видеокодирования представляет одну из наиболее сложных инженерных задач современности. Каждый тип сети оптимален для определенных аспектов видеоданных: CNN превосходно обрабатывают пространственные корреляции, RNN эффективны для временных зависимостей, а трансформеры способны учитывать долгосрочный контекст.

Особый интерес представляют гибридные архитектуры, где традиционные методы кодирования комбинируются с ML-компонентами для решения специфических задач - например, использование GAN для пост-обработки артефактов сжатия или применение attention-механизмов для улучшения предсказания движения в сложных сценах.

2.3.3 Квантовые алгоритмы

Квантовые вычисления могут революционизировать видеокодирование в будущем:

  1. Квантовое преобразование Фурье: более эффективные трансформации
  2. Квантовый поиск: оптимизация выбора режимов кодирования
  3. Квантовая оптимизация: решение задач rate-distortion
  4. Временные рамки: практическое применение через 10-15 лет
  5. Исследования: IBM, Google, Microsoft ведут активные работы

Квантовые вычисления обещают фундаментально изменить подход к видеокодированию через способность решать определенные классы оптимизационных задач экспоненциально быстрее классических компьютеров. В контексте видеокодирования это может означать революцию в области rate-distortion оптимизации и поиска оптимальных режимов кодирования.

Однако практическое применение квантовых алгоритмов в видеокодировании потребует не только развития квантового аппаратного обеспечения, но и создания гибридных систем, где квантовые процессоры будут работать в тандеме с классическими. Наиболее перспективными выглядят применения для оптимизации параметров кодирования и анализа сложных видеосцен с множественными объектами.

Будущее видеокодирования лежит в интеграции традиционных алгоритмов сжатия с методами искусственного интеллекта. Это позволит достичь беспрецедентной эффективности при сохранении вычислительной доступности для широкого круга устройств.
